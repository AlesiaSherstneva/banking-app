# banking-app

Для запуска приложения нужно запустить его в среде разработки. Также должен быть
запущен `Docker`, так как в нём разворачивается база данных.

## API Endpoints

Для удобства проверки работы приложения я добавила экспортированную из `Postman` коллекцию 
запросов, с помощью которой я тестировала функционал в процессе разработки (файл 
`banking-app.postman_collection.json`). Её можно импортировать обратно в `Postman`.

### Аутентификация

| Метод | Эндпоинт          | Описание                     | Требуемая авторизация |
|-------|-------------------|-----------------------------|-----------------------|
| POST  | `/api/login` | Вход в систему (получение JWT) | Нет                   |

### Пользователи

| Метод | Эндпоинт       | Описание                          | Требуемая авторизация |
|-------|----------------|----------------------------------|-----------------------|
| GET   | `/api/users`   | Поиск пользователей с фильтрами   | Нет                   |
| GET   | `/api/users/{id}` | Получить данные пользователя по ID | Нет                   |

### Управление контактами

| Метод | Эндпоинт            | Описание                          | Требуемая авторизация |
|-------|---------------------|----------------------------------|-----------------------|
| GET   | `/api/me`      | Получить свой профиль             | Да (JWT)              |
| PATCH | `/api/users/me/emails`  | Добавить/изменить/удалить email  | Да (JWT)              |
| PATCH | `/api/users/me/phones`  | Добавить/изменить/удалить телефон | Да (JWT)              |

### Банковские операции

| Метод | Эндпоинт       | Описание                     | Требуемая авторизация |
|-------|----------------|-----------------------------|-----------------------|
| POST  | `/api/transfer` | Выполнить перевод средств   | Да (JWT)              |

## Тестовые учетные данные

Получить JWT можно при помощи эндпоинта `/api/login`, передав в него учётные данные одного из
трёх зарегистрированных пользователей.

| Email               | Пароль      |
|---------------------|-------------|
| ivanov@example.com  | ivanov123   |
| petrova@example.com | petrova123  |
| sidorov@example.com | sidorov123  |

Полученный токен следует передать в запросе, требующем JWT, в заголовке (header) "Authorization",
прибавив к нему строку "Bearer ".

Либо, если запрос производится из `Postman`, можно во вкладке "Authorization", выбрать "Auth Token" ->
"Bearer token", а в поле "Token" ввести токен (не прибавляя "Bearer ").
